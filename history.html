HTMLs Historia

Under de första fem åren (1990-1995) gick HTML igenom ett antal revisioner och fick ett antal tillägg, främst värd först på CERN och sedan på IETF.

Med skapandet av W3C ändrades HTMLs utvecklingsplats igen. Ett första misslyckat försök att utöka HTML 1995, känd som HTML 3.0, gjorde sedan plats för en mer praktisk metod känd som HTML 3.2, som slutfördes 1997. HTML4 följde snabbt senare samma år.

Året därpå beslutade W3C-medlemskapet att sluta utveckla HTML och istället börja arbeta på en XML-baserad motsvarighet, kallad XHTML. Detta försök började med en omformulering av HTML4 i XML, känd som XHTML 1.0, som inte lade till några nya funktioner förutom den nya serialiseringen, och som slutfördes 2000. Efter XHTML 1.0 vändes W3C:s fokus till att göra det lättare för andra arbetsgrupper att utöka XHTML, under flaggan XHTML Modularization. Samtidigt med detta arbetade W3C också på ett nytt språk som inte var kompatibelt med de tidigare HTML- och XHTML-språken, kallar det XHTML2.

Runt den tid då HTMLs utveckling stoppades 1998 specificerades delar av API:et för HTML utvecklat av webbläsarleverantörer och publicerades under namnet DOM Level 1 (1998) och DOM Level 2 Core och DOM Level 2 HTML (med start 2000 och som kulminerade 2003). Dessa ansträngningar försvann sedan, med några DOM nivå 3-specifikationer publicerade 2004 men arbetsgruppen stängdes innan alla nivå 3-utkast var färdiga.

Hur fungerar webben och vad är DOM?

DOM, eller Document Object Model, är ett programmeringsgränssnitt för HTML- och XML-dokument. Det låter utvecklare manipulera och hantera innehållet i en webbsida på ett strukturerat sätt. T.ex. kan man använda DOM för att ändra stilen på ett element på en webbsida, eller för att lägga till nya element. DOM fungerar som en slags "trädstruktur" av alla element i ett dokument, vilket gör det enkelt att navigera och ändra på dessa element.

Detaljerat sett är DOM en abstraktion av ett HTML- eller XML-dokument, där varje element i dokumentet representeras som en objektnode i en trädstruktur. Varje objektnode har egenskaper och metoder som utvecklare kan använda för att manipulera innehållet i dokumentet. Så när en webbläsare läser in en HTML-sida, skapar den ett DOM-objekt som en representation av sidans innehåll. Utvecklare kan sedan använda JavaScript eller ett annat programmeringsspråk för att interagera med DOM-objektet och på så sätt ändra sidans innehåll. Till exempel kan man använda DOM för att lägga till ett nytt element till en webbsida, ändra stilen på ett befintligt element, eller ta bort ett element helt. Detta kan göras genom att navigera i trädstrukturen och anropa olika metoder på objektnode-objekten. I korthet är DOM ett sätt att strukturera och manipulera innehållet i en webbsida på ett programmerbart sätt. Det gör det möjligt för utvecklare att skapa dynamiska och interaktiva webbplatser.
